# Точечное исправление runtime-регрессий Яндекс Игр

## Цель

Устранить три подтверждённые проблемы production-сборки без изменения игровой механики и без широкого рефакторинга:

1. повторяющиеся `404` при загрузке графики комнат;
2. превышение лимита вызовов `player.setData()`;
3. перенос текста по одной букве в узкой desktop-колонке.

Дополнительно расширить локальный аудит Яндекс Игр, чтобы такая ошибка пути обнаруживалась до загрузки архива.

## Исправление путей ресурсов

Все динамические пути к изображениям комнат, наград и кота должны формироваться относительно `import.meta.env.BASE_URL`, который при `base: './'` указывает на каталог HTML-файла игры. В production-коде не должно остаться локальных путей, начинающихся с `/assets/`.

Если загрузка конкретного ресурса завершилась ошибкой, его alias остаётся в наборе недоступных ресурсов. Повторная отрисовка того же состояния не должна немедленно запускать тот же запрос снова. Процедурная сцена остаётся резервным отображением.

## Ограничение облачных сохранений

Локальное сохранение остаётся немедленным при каждом изменении игрового состояния. Облачное сохранение получает последнюю сериализованную версию состояния и объединяет частые изменения в один запрос не чаще одного раза в 15 секунд.

Объект `Player` кешируется после первого успешного `sdk.getPlayer()`, поэтому загрузка и последующие сохранения не расходуют лимит инициализации игрока. Одновременно выполняется не более одного облачного сохранения; если состояние изменилось во время запроса, после разрешённого интервала отправляется последняя версия.

При переходе страницы в скрытое состояние ожидающее облачное сохранение запускается сразу. Ошибки облака остаются безопасными: локальная копия уже записана и игра продолжает работу.

## Узкая desktop-колонка

В карточках модулей предпросмотр действия перестаёт делить одну строку между текстом и неразрывными тегами. Для предпросмотра внутри `.module-panel` теги размещаются следующей строкой и могут переноситься. Остальные варианты `ActionPreviewLine` не меняются.

Это устраняет буквенный перенос при ширине около desktop-порога, сохраняя текущую структуру desktop- и mobile-layout.

## Аудит

Проверка production-артефакта должна анализировать JavaScript-бандлы и выдавать ошибку для локальных root-relative ссылок `/assets/...`. Ссылки SDK и внешние абсолютные URL не относятся к этой проверке.

## Проверка

- Unit-тесты подтверждают относительные пути всех динамических игровых ресурсов и отсутствие повторной загрузки недоступного alias.
- Тест hook-а подтверждает немедленное локальное сохранение и ограниченное количество облачных вызовов при секундных тиках.
- CSS-регрессия подтверждает отдельную строку и перенос тегов в карточках модулей.
- Тест аудитора подтверждает обнаружение `/assets/...` внутри production-JS.
- После исправлений выполняются полный `npm test`, release-сборка, проверка ZIP и аудит Яндекс Игр.

## Вне области изменений

- игровая экономика и частота секундного тика;
- формат данных сохранения;
- общий редизайн карточек или изменение breakpoint;
- изменение правил и снимка требований Яндекс Игр.
