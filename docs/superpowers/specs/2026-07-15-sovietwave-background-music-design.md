# Фоновая sovietwave-музыка

## Цель

Добавить в «Космическую коммуналку» оригинальную ненавязчивую фоновую музыку для продолжительных игровых сессий. Музыка должна передавать настроение советской космической фантастики и мягкого sovietwave, не копируя мелодии или аранжировки ППВК либо других исполнителей.

## Звучание

- Инструментальная композиция без вокала.
- Темп около 80–90 BPM.
- Тёплые аналоговые синтезаторные тембры, мягкая драм-машина, спокойная басовая линия и простая меланхоличная космическая мелодия.
- Низкая музыкальная плотность, чтобы трек не мешал интерфейсным звукам и не утомлял при повторении.
- Бесшовный цикл с плавным входом и выходом.

## Архитектура

Музыка становится частью существующего модуля `src/platform/sound.ts`, который уже отвечает за эффекты, mute-состояние и паузу при скрытии страницы. Публичный интерфейс модуля расширяется функциями запуска и остановки фоновой музыки. `useGameState` управляет её жизненным циклом тем же способом, которым сейчас управляет гулом станции.

Фоновая композиция создаётся процедурно средствами Web Audio API. Это сохраняет компактный размер сборки для Яндекс Игр, исключает внешние лицензируемые аудиофайлы и позволяет сделать цикл без паузы на границе файла.

## Поведение

- Музыка запускается, когда игра готова и звук не выключен.
- Из-за ограничений браузерного autoplay фактическое воспроизведение может начаться после первого действия пользователя.
- Существующая кнопка звука одновременно выключает музыку, гул и эффекты; отдельная настройка громкости в эту задачу не входит.
- При выключении звука музыка плавно затихает и освобождает созданные аудиоузлы.
- При повторном включении музыка запускается заново с плавным входом.
- При скрытии вкладки общий `AudioContext` приостанавливается, а при возвращении возобновляется, сохраняя требования Яндекс Игр.
- Интерфейс и тексты не меняются.

## Сведение

Музыка звучит тише интерфейсных эффектов. Существующий гул реактора остаётся нижним атмосферным слоем, но его уровень должен быть достаточно низким, чтобы он не маскировал бас и основную мелодию.

## Отказоустойчивость

При отсутствии Web Audio API или ошибке создания аудиоузлов игра продолжает работать без музыки. Повторный запуск и остановка идемпотентны: повторный вызов запуска не создаёт второй трек, а повторная остановка безопасна.

## Проверка

- Unit-тесты подтверждают безопасный запуск и остановку без `AudioContext`.
- Тесты с подменным Web Audio API подтверждают отсутствие дублирующего запуска, остановку аудиоузлов и соблюдение mute-состояния.
- Существующие тесты звуковых эффектов продолжают проходить.
- Production-сборка проходит проверку TypeScript и Vite.
- Ручная проверка подтверждает комфортную громкость, отсутствие заметного шва в цикле и корректную реакцию на mute и сворачивание вкладки.

## Вне объёма

- Вокал и заимствованные музыкальные темы.
- Несколько треков, плейлист или музыка, меняющаяся между комнатами.
- Отдельные регуляторы громкости музыки и эффектов.
- Загрузка аудиофайлов из сети.
