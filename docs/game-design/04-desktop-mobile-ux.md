# Desktop And Mobile UX

## Общая архитектура интерфейса

Игра использует одну экономику и один state. Layout выбирается по размеру экрана:

- mobile: вертикальный layout, нижняя навигация, крупные карточки;
- desktop: трехколоночный layout, больше данных одновременно.

Порог для desktop layout: ширина от 900 px. Ниже используется mobile/tablet layout.

Визуальный стиль UI следует документу [07-visual-style.md](07-visual-style.md): теплые панели, мягкие контрасты, округлые формы, ламповые индикаторы и читаемые пиктограммы. Ретро-детали не должны ухудшать скорость чтения чисел и кнопок.

Станция рендерится отдельной PixiJS-сценой внутри React layout. React не должен переносить кнопки, вкладки и длинный текст в canvas: интерактивный UI остается DOM, а canvas отвечает за атмосферу, рост станции и короткие визуальные реакции на покупки.

## Mobile layout

Mobile - основной режим короткой сессии.

Структура экрана:

```text
[верхняя панель: кредиты, доход/сек]
[центр: станция и быстрый бонус]
[нижняя зона: текущая вкладка]
[нижняя навигация: Комнаты | Апгрейды | Бонусы | Реновация]
```

Требования:

- все основные кнопки не меньше 44 px по высоте;
- покупка уровня доступна в один тап;
- длинные числа сокращаются: `12.4K`, `8.1M`, `3.2B`;
- карточка модуля показывает цену, доход и следующий milestone;
- рекламная кнопка не перекрывает покупку;
- при возврате в игру офлайн-награда показывается до основного экрана.

## Desktop layout

Desktop - режим контроля и сравнения.

Структура экрана:

```text
[верхняя панель: кредиты, доход/сек, комфорт, репутация]
[левая колонка: модули и уровни]
[центр: визуальная станция]
[правая колонка: цели, жильцы, бонусы]
[нижняя строка: вкладки/фильтры и служебные кнопки]
```

Требования:

- игрок видит минимум 5 модулей без прокрутки;
- цели и бонусы не скрывают станцию;
- наведение мышью показывает краткую подсказку;
- кнопки покупки имеют состояния `доступно`, `недостаточно кредитов`, `max milestone`;
- визуальная станция остается главным фокусом, а не фоном.

## Состояния интерфейса

Обязательные состояния:

- первый запуск;
- загрузка сохранения;
- обычная игра;
- покупка доступна;
- покупка недоступна;
- офлайн-награда;
- rewarded ad доступна;
- rewarded ad недоступна;
- ошибка SDK или сохранения;
- prestige подтверждение;
- после prestige.

## Desktop/mobile различия

| Аспект | Mobile | Desktop |
|---|---|---|
| Навигация | нижние табы | колонки и вкладки |
| Станция | крупно по центру | центр с боковыми панелями |
| Модули | карточки по одной колонке | список/таблица |
| Подсказки | короткий текст в карточке | hover/tooltips |
| Реклама | отдельная вкладка и бонус-кнопки | правая колонка бонусов |
| Prestige | отдельный экран | правая панель + модал подтверждения |

## Первый экран

На первом экране игрок должен видеть:

- станцию;
- текущие кредиты;
- доход в секунду;
- первую доступную покупку;
- понятную цель "Заселить первого жильца".

Нельзя начинать игру с длинного обучения. Первые подсказки должны быть встроены в действия.

Первый экран должен сразу сообщать стиль: старая орбитальная коммуналка с теплым окном, табличкой станции и первым жилым модулем. Даже до первой покупки игрок должен понимать, что это уютный космический дом, а не обычный абстрактный кликер.

PixiJS-сцена должна иметь стабильные размеры в обоих layout: canvas не должен менять высоту при покупке модулей, переключении вкладок или появлении бонусов.

## Clarity Spine

Every major action surface should answer four questions in one compact place:

- what matters now;
- why it matters;
- what changes after the action;
- where the result is visible.

Desktop can show a short reason line and tags. Mobile should keep the same
preview data but render it as compact tags plus one result line.
