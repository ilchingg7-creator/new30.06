# Yandex Games Integration

## Цель документа

Этот файл фиксирует требования, которые нужно учитывать при реализации под Yandex Games. Это не публикационный аудит и не гарантия прохождения модерации. Формальный аудит нужно проводить по готовому билду.

## SDK

Игра должна использовать Yandex Games SDK для:

- инициализации платформы;
- сообщения о готовности загрузки;
- сохранений;
- rewarded ads;
- interstitial ads, если они будут включены;
- локализации, если проект расширится за русский язык.

После загрузки основного экрана игра должна вызвать готовность платформы через SDK. До этого игрок не должен видеть поломанный или пустой экран.

## Сохранения

Сохранять:

- кредиты;
- суммарно заработанные кредиты;
- уровни модулей;
- открытые модули;
- комфорт;
- репутацию станции;
- купленные prestige-улучшения;
- открытых жильцов;
- время последнего сохранения;
- активные временные бонусы и их окончание.

Сохранение должно работать локально как fallback, даже если облачное сохранение недоступно.

## Офлайн-доход

Офлайн-доход считается по времени последнего сохранения.

MVP-правило:

```text
офлайн-доход = доход/сек * min(время отсутствия, 8 часов)
```

Если активный рекламный множитель истек во время отсутствия, он не должен применяться ко всему офлайн-периоду. Для MVP можно не учитывать временные бонусы в офлайн-доходе, чтобы избежать спорных расчетов.

## Реклама

Rewarded ads должны быть добровольными. Награда выдается только после успешного завершения просмотра.

Минимальный набор:

- удвоение офлайн-дохода;
- x2 доход на 5 минут;
- VIP-жилец на 10 минут.

Interstitial ads в MVP можно отключить. Если добавить позже, правила:

- не показывать в первые 3 минуты;
- не показывать чаще одного раза в 3-5 минут;
- не показывать после каждого действия;
- не показывать перед выдачей награды.

## Загрузка

Требования к ощущению загрузки:

- первый экран не пустой;
- есть индикатор загрузки;
- assets подгружаются до старта;
- после готовности игрок сразу видит станцию и первую покупку.

## Адаптивность

Проверяемые размеры:

- mobile portrait: 360x640;
- mobile portrait: 390x844;
- tablet portrait: 768x1024;
- desktop: 1366x768;
- desktop wide: 1920x1080.

Игра должна сохранять читаемость текста и доступность кнопок на каждом размере.

## Риски публикации

- слишком тяжелый билд;
- пустой экран до загрузки SDK;
- поломка при смене фокуса вкладки;
- неработающие сохранения;
- рекламная награда выдается без успешного просмотра;
- текст не помещается на mobile;
- игра не продолжает прогресс после возвращения из рекламы;
- нет понятного первого действия.

## Минимальная проверка перед публикацией

- новый игрок проходит первые 5 минут без ошибок;
- сохранение восстанавливается после перезагрузки страницы;
- офлайн-доход начисляется после паузы;
- rewarded ad корректно дает награду;
- игра не ломается при сворачивании и возвращении;
- desktop и mobile layouts не перекрывают текст и кнопки;
- билд укладывается в лимиты платформы.
