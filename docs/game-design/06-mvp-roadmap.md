# MVP Roadmap

## MVP scope

MVP должен доказать, что cozy-comedy idle loop работает на desktop и mobile. В MVP не нужны сложная сюжетная кампания, десятки жильцов, магазин косметики и полноценные сезонные события.

MVP включает:

- 8 модулей станции;
- 6 жильцов;
- 20-30 апгрейдов;
- 10-15 целей;
- офлайн-доход;
- rewarded ads;
- prestige через "Реновацию орбиты";
- responsive desktop/mobile UI;
- PixiJS station scene embedded inside React UI;
- сохранения через Yandex Games SDK и local fallback.

## Этап 1: Игровая модель

Цель: сделать экономику без UI-зависимостей.

Результат:

- структура состояния игрока;
- расчет дохода/сек;
- покупка модулей и уровней;
- milestone множители;
- расчет офлайн-дохода;
- prestige reset;
- сериализация сохранения.

Критерий готовности: экономику можно тестировать из кода без браузера.

## Этап 2: Первый playable экран

Цель: игрок может зарабатывать и покупать улучшения.

Результат:

- главный экран;
- PixiJS-сцена станции;
- список модулей;
- кнопка покупки;
- текущие кредиты и доход/сек;
- базовая адаптивность.

Критерий готовности: первые 5 минут игры проходят без ручных правок состояния.

## Этап 3: Desktop/mobile layouts

Цель: две версии интерфейса на одной кодовой базе.

Результат:

- mobile layout с нижними табами;
- desktop layout с боковыми панелями;
- стабильный canvas-контейнер станции в обоих layout;
- корректные состояния кнопок;
- форматирование больших чисел;
- отсутствие наложений текста.

Критерий готовности: игра читается на 390x844 и 1366x768.

## Этап 4: Контент MVP

Цель: наполнить loop достаточным количеством целей.

Результат:

- 8 модулей;
- 20-30 апгрейдов;
- 6 жильцов;
- 10-15 целей;
- визуальные этапы станции.

Критерий готовности: игрок видит новые открытия минимум первые 30 минут, а последний модуль не открывается за первые 2-3 минуты нормальной игры.

## Этап 5: Сохранения и Yandex Games

Цель: подготовить платформенную основу.

Результат:

- SDK initialization;
- loading ready event;
- cloud save;
- local save fallback;
- offline reward screen;
- pause/resume handling.

Критерий готовности: прогресс сохраняется после перезагрузки и возвращения в игру.

## Этап 6: Rewarded ads

Цель: добавить добровольную монетизацию без давления.

Результат:

- x2 доход на 5 минут;
- удвоение офлайн-дохода;
- VIP-жилец на 10 минут;
- корректная выдача награды только после успешного просмотра.

Критерий готовности: отказ от рекламы не блокирует прогресс.

## Этап 7: Баланс и публикационный пакет

Цель: довести MVP до тестовой публикации.

Результат:

- баланс первых 60 минут;
- иконка;
- обложка;
- 3-5 скриншотов;
- описание;
- теги;
- проверка desktop/mobile;
- проверка размера билда.

Критерий готовности: новый игрок понимает игру без подсказок разработчика.

## Порядок разработки

Рекомендуемый порядок:

1. Экономика.
2. Минимальный UI.
3. Responsive layout.
4. Контент.
5. Сохранения.
6. Yandex SDK.
7. Реклама.
8. Баланс.
9. Полировка.

Такой порядок снижает риск: сначала проверяется игровой loop, затем платформа и монетизация.
